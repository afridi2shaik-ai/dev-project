## File Format Standards

### Metadata Headers
Each document starts with:
```markdown
# Title

> 🎯 **Subtitle** • Description
```

### Navigation Footer
Each document ends with:
```markdown
## See Also

- [README.md](README.md) - Documentation index

---

📖 **Return to**: [README.md](README.md)
```

### Code Examples
- Use language tags: `python`, `bash`, `json`, `sql`, etc.
- Include both code and explanation
- Show example inputs and outputs
- Reference related sections

### Diagrams
- Use mermaid for diagrams
- Follow mermaid syntax rules
- Keep diagrams focused
- Include captions/labels

---

## Maintenance Guidelines

### When Adding New Docs
1. Follow naming convention: `TOPIC.md`
2. Add metadata header with emoji
3. Cross-reference from README.md
4. Update FILE_STRUCTURE.md
5. Add footer with See Also section

### When Updating Docs
1. Keep cross-references current
2. Update version in README.md if significant
3. Test all links work
4. Verify code examples still work
5. Update table of contents if needed

### Keeping Links Working
- Use relative paths: `[Doc](Doc.md)`
- Don't use absolute URLs for internal docs
- Update links when moving files
- Test links periodically
